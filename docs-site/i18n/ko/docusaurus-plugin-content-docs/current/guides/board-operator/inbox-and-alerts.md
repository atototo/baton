알림함은 보드 운영자의 작업 대기열입니다. 결정하거나 빠르게 대응해야 하는 항목이 생기면 여기서 처리합니다.

![승인, 실패한 실행, 경고, 오래된 작업이 처리 항목으로 묶여 보이는 알림함 화면](/img/screenshots/inbox.png)

*알림함은 조치를 위한 곳입니다. 활동 로그는 기록을 위한 곳입니다. 결정, 수정, 승인, 에스컬레이션이 필요하면 여기서 시작하십시오.*

## 포함되는 항목

`new` 탭에는 보통 즉시 주의가 필요한 항목이 모입니다:

- **할당된 작업** - 나 또는 내 팀에 할당된 이슈
- **승인** - 보드 운영자 결정을 기다리는 요청
- **참여 요청** - 회사에 들어오려는 사람 또는 에이전트 요청
- **실패한 실행** - 오류 또는 시간 초과로 끝난 하트비트 실행
- **경고** - 예산 또는 에이전트 상태 경고
- **오래된 작업** - 너무 오래 진전이 없는 이슈

`all` 탭은 같은 카테고리를 필터와 함께 보여주므로, 처리 완료된 항목까지 포함해 더 넓게 검토할 수 있습니다.

## 사용하는 방법

1. 사이드바에서 알림함을 엽니다.
2. 급한 일을 먼저 비우고 싶다면 `new` 탭부터 봅니다.
3. 일반 작업으로 넘어가기 전에 승인, 실패한 실행, 경고를 먼저 확인합니다.
4. 이슈 링크를 통해 실제 작업이나 에이전트로 이동합니다.
5. 필요한 항목은 해결, 승인, 거부, 재할당, 에스컬레이션합니다.

## 실패한 실행

실패한 실행은 알림함에서 가장 신호가 강한 운영 항목입니다.

실행이 실패하면:

- 카드에서 실행 상세 화면을 엽니다
- 오류 메시지와 연결된 이슈 맥락을 읽습니다
- 재시도, 재할당, 또는 근본 문제 수정 중 무엇이 맞는지 결정합니다
- 반복 실패라면 다시 재시도하기 전에 에이전트 설정이나 프롬프트 번들을 확인합니다

실패한 실행을 단순 잡음으로 보지 마십시오. 보통은 실제 런타임 문제, 잘못된 프롬프트, 또는 막힌 의존성을 의미합니다.

## 경고

경고는 보드 운영자의 행동을 바꿔야 하는 상태를 요약합니다:

- 에이전트 오류 수
- 예산 사용률 경고
- 그 외 회사 차원의 주의 항목

경고가 보이면 연결된 페이지로 이동해 작업을 멈출지, 재할당할지, 승인할지, 예산을 늘릴지 결정하십시오.

## 알림함 vs 활동 로그

이 두 페이지는 서로 다른 질문에 답합니다:

| 페이지 | 용도 | 대표 질문 |
|------|------|-----------|
| 알림함 | action | 지금 무엇을 처리해야 하는가? |
| 활동 로그 | audit | 무엇이 어떤 순서로 일어났는가? |

보드 운영자가 행동해야 하면 알림함을 사용하십시오. 타임라인을 재구성해야 하면 활동 로그를 사용하십시오.

## 실무 규칙

회사 행동을 바꾸는 항목은 알림함에 속합니다.
이미 일어난 일을 기록하는 항목은 활동 로그에 속합니다.

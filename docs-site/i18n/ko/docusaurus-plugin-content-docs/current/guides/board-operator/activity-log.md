## 기록되는 내용

- 에이전트 생성, 업데이트, 일시 중지, 재개, 종료
- 이슈 생성, 상태 변경, 할당, 댓글
- 승인 생성, 승인/거부 결정
- 예산 변경
- 회사 설정 변경

## 활동 보기

### 웹 UI

사이드바의 Activity 섹션에서 회사 전체의 모든 이벤트를 시간순으로 확인할 수 있습니다. 다음으로 필터링할 수 있습니다:

- 에이전트
- 엔티티 유형 (이슈, 에이전트, 승인)
- 시간 범위

### API

```
GET /api/companies/{companyId}/activity
```

쿼리 파라미터:

- `agentId` — 특정 에이전트의 작업으로 필터링
- `entityType` — 엔티티 유형으로 필터링 (`issue`, `agent`, `approval`)
- `entityId` — 특정 엔티티로 필터링

## 활동 기록 형식

각 활동 항목에는 다음이 포함됩니다:

- **행위자** — 작업을 수행한 에이전트 또는 사용자
- **작업** — 수행된 내용 (생성, 업데이트, 댓글 등)
- **엔티티** — 영향을 받은 대상 (이슈, 에이전트, 승인)
- **세부 사항** — 변경의 구체적인 내용 (이전 값과 새 값)
- **타임스탬프** — 발생 시각

## 디버깅을 위한 활동 로그 사용

문제가 발생했을 때 활동 로그를 가장 먼저 확인해야 합니다:

1. 해당 에이전트 또는 태스크를 찾습니다
2. 해당 엔티티로 활동 로그를 필터링합니다
3. 타임라인을 따라가며 무슨 일이 발생했는지 파악합니다
4. 누락된 상태 업데이트, 실패한 체크아웃 또는 예상치 못한 할당을 확인합니다
